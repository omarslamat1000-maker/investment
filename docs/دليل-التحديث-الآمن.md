# دليل التحديث الآمن — كيف تضيف ميزة دون كسر الإنتاج

## دورة إضافة أي ميزة جديدة

```bash
# 1) فرع جديد من main — لا تعمل على main مباشرة
git checkout main && git pull
git checkout -b feature/اسم-الميزة

# 2) التطوير محلياً مع المعاينة الحية
npx http-server -p 8123
# افتح http://localhost:8123 وجرّب يدوياً

# 3) الاختبار الآلي قبل الالتزام
for f in js/*.js; do node --check "$f"; done
node -e "global.window={};require('./data/lands-data.js');console.log(window.DEFAULT_LANDS_GEOJSON.features.length+' قطعة')"

# 4) الالتزام والدفع للمراجعة
git add . && git commit -m "وصف واضح للميزة"
git push -u origin feature/اسم-الميزة
# افتح Pull Request على GitHub — خط CI يفحص الفرع تلقائياً

# 5) الدمج والنشر
# بعد المراجعة: Merge إلى main ← الفحص والنشر تلقائيان
# حدّث CHANGELOG.md برقم إصدار جديد (فرعي للميزات، إصلاح للأخطاء)
```

## قواعد السلامة الخاصة بهذه المنصة

1. **لا تغيّر بنية مفاتيح التخزين** (`edits`, `sections`, `settings`, `userLandmarks`, `landmarkOverrides`) دون كود ترحيل — بيانات المستخدمين محفوظة في متصفحاتهم بالبنية الحالية، وأي تغيير كاسر يفقدهم بياناتهم. عند الاضطرار: اقرأ البنية القديمة وحوّلها ثم اكتب الجديدة، وارفع الرقم الرئيسي في الإصدار.
2. **أي كتابة تخزين جديدة**: فورية (بلا تأجيل) + بثّ عبر BroadcastChannel المناسبة حتى تتزامن التبويبات.
3. **أي محتوى جديد في تقرير الطباعة**: تحقق أن الصفحة العرضية الواحدة ما زالت تتسع (المستهلَك حالياً ~161مم من 194مم).
4. **وسوم الخرائط الدائمة**: استخدم نمط divIcon المرتكز — تلميحات Leaflet تنزاح في RTL.
5. **جرّب دائماً بحالة بيانات نظيفة أيضاً**: ⚙ ← «مسح جميع البيانات والعودة للافتراضي» ثم أعد التجربة.

## خطة التراجع عند فشل نشر

```bash
git revert HEAD && git push   # يعيد النسخة السابقة تلقائياً عبر خط النشر
```
بيانات المستخدمين لا تتأثر بالنشر أو التراجع (محلية في متصفحاتهم).

## الصيانة الدورية

| الدورية | المهمة |
|---|---|
| شهرياً | مراجعة إصدارات مكتبات CDN (Leaflet/Turf/SheetJS/Chart.js) وتحديث الروابط المثبتة عند صدور إصلاحات أمنية |
| شهرياً | تجربة المسارات الحرجة يدوياً بعد أي تحديث متصفح رئيسي (رفع ملف، طباعة، حفظ/استعادة نسخة) |
| ربع سنوياً | التحقق من استمرار عمل الخدمات العامة (OSRM للتوجيه، Nominatim للبحث) وبدائلها عند الحاجة |
| عند تحديث بيانات الأمانة | `node tools/convert-kml.js doc.kml data/lands-data.js` ثم فحص CI محلياً قبل الدفع |
| بعد كل نشر | فتح الرابط المنشور والتحقق من التحميل والكونسول نظيف |
